Мы разрабатываем платформу для фрилансеров (биржу заказов), где заказчики публикуют задания, а исполнители откликаются на них. 
Бизнес-требование: Роли заказчика и исполнителя объединены в один аккаунт (User). Разделение контекста происходит на уровне интерфейса личного кабинета. Авторизация строго через GitHub OAuth.

Технологический стек проекта:
├── Ядро: Go (Golang 1.22+) + Роутер Chi v5
├── Фронтенд: HTMX v2 + gomponents (UI на чистом Go) + Tailwind CSS (через native CLI)
├── Клиентский интерактив: Alpine.js (подключается через CDN скриптом для модалок/меню без участия сервера)
├── База данных: PostgreSQL + GORM
├── Авторизация и сессии: GitHub OAuth + alexedwards/scs/v2 + драйвер alexedwards/scs/redisstore
├── Кэш/Хранилище сессий: Valkey (через go-redis/v9 клиент, совместимый с Redis-протоколом)
├── Безопасность: gorilla/csrf
└── Dev-инструменты: Air (live reload для Go)

Твоя задача — инициализировать проект, подготовить зависимости, настроить окружение и собрать рабочий каркас приложения согласно Standard Go Project Layout.

Выполни следующие шаги:

1. Создай Docker Compose (`docker-compose.yml`)
- Подними контейнер PostgreSQL (версия 16+) с персистентным volume.
- Подними контейнер Valkey (версия 7.2+ или последняя стабильная) для сессий.
- Настрой проброс портов и переменные окружения для подключения бэкенда.

2. Инициализируй Go-модуль и зависимости
- Сгенерируй `go.mod` и добавь актуальные версии пакетов: go-chi/chi/v5, gorm.io/gorm, gorm.io/driver/postgres, alexedwards/scs/v2, alexedwards/scs/redisstore, redis/go-redis/v9, gorilla/csrf, maragudk/gomponents.

3. Создай структуру проекта
Организуй код по следующим директориям:
├── cmd/server/main.go (точка входа)
├── internal/config/ (загрузка .env переменных: порты, бд, github oauth credentials)
├── internal/models/ (GORM модели: User, Order, Bid с правильными связями)
├── internal/web/ (роутер Chi, middleware для логирования, восстановления, сессий и CSRF)
├── internal/web/handlers/ (заглушки для auth, профиля, заказов)
├── internal/ui/ (базовый layout и компоненты на gomponents с подключенным Tailwind, HTMX и Alpine.js)
└── ui/ (директория для сгенерированных статических файлов: глобальный style.css)

4. Настрой Dev-пайплайн
- Напиши конфигурационный файл для Air (`.air.toml`), чтобы он перезапускал `cmd/server/main.go` при изменении `.go` файлов.
- Создай `Makefile` с командами: 
  * `make dev` (запуск Air для бэкенда)
  * `make tailwind-watch` (команда запуска автономного бинарника tailwindcss для сканирования internal/ui и сборки стилей в ui/style.css)
  * `make up` (запуск docker-compose)

5. Подготовь базовый код инициализации в `main.go`
- Подключение к PostgreSQL через GORM и запуск AutoMigrate для моделей.
- Подключение к Valkey через `go-redis`.
- Инициализация `scs.New()` с использованием `redisstore.New(valkeyClient)`.
- Настройка роутера Chi с подключением сессий и базовым выводом главной страницы через `gomponents`.

Сгенерируй файлы конфигурации, docker-compose, структуру и инициализационный Go-код. Пиши чистый, идиоматичный Go-код без скрытых заглушек (без "// TODO: implement").
